## Dev Notes

### Current Implementation Locations

**Operator Controller**: `app/Http/Controllers/OperatorController.php`

**Routes Definition**: `routes/api.php` (lines 17-23)
```
Route::post('/entry', [OperatorController::class, 'entry'])->name('entry');
Route::middleware(['internalCheck'])->name('internal.')->prefix('internal')->group(function () {
    Route::name('operator.')->prefix('operator')->group(function () {
        Route::get('/', [OperatorController::class, 'list'])->name('list');
        Route::post('/store', [OperatorController::class, 'store'])->name('store');
        Route::delete('/delete/{operator:slug}', [OperatorController::class, 'delete'])->name('delete');
    });
});
```

**Health Check Route**: `routes/api.php` (lines 26-28)
```
Route::get('/status', function () {
    return response()->json(['message' => 'OK'], 200);
});
```

**Image Controller**: `app/Http/Controllers/ImageController.php` (web routes & dashboard)

### Key Components
- **Operator Model**: `app/Models/Operator.php` (CRUD operations)
- **internalCheck Middleware**: `app/Http/Middleware/` (access control)
- **Image Processing**: Uses intervention/image package

### Admin Dashboard
- Dashboard controllers: `app/Http/Controllers/Dashboard/`
- Routes: `routes/web.php` (lines 15-19)
```
Route::name('dashboard.')->middleware('auth')->group(function () {
    Route::get('/dashboard', [ImageController::class, 'index'])->name('index');
    Route::get('/dashboard/edit/{id}', [ImageController::class, 'edit'])->name('edit');
    Route::put('/dashboard', [ImageController::class, 'update'])->name('update');
});
```

### Admin API vs Dashboard
- **Admin API**: Programmatic operator management via `/v1/internal/operator/*`
- **Admin Dashboard**: Web UI for management via `routes/web.php`
- Both access same Operator model but via different controllers

### Go Implementation Considerations
1. **Operator Management**: Simple CRUD with slug-based access
2. **Authentication**: Admin middleware pattern (check operator/user)
3. **Image Upload**: File handling and storage (S3? Local disk?)
4. **Health Check**: Return basic OK response (can be expanded for database checks)
